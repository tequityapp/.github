# Contributing to Tequity

Thanks for taking the time to contribute. 🙏

These are the default contribution guidelines for all
[tequityapp](https://github.com/tequityapp) repositories. An individual project
may add its own `CONTRIBUTING.md`, which takes precedence.

## Code of Conduct

This project and everyone participating in it is governed by our
[Code of Conduct](./CODE_OF_CONDUCT.md). By participating, you are expected to
uphold it.

## How can I contribute?

### Reporting bugs

- Search existing issues first to avoid duplicates.
- Open a new issue using the **Bug Report** template and fill in as much detail
  as you can.

### Suggesting enhancements

- Open an issue using the **Feature Request** template.
- Explain the problem you're trying to solve, not just the solution you have in
  mind.

## Development workflow

1. **Branch** from the default branch.
2. **Make your changes** in small, focused commits with clear messages.
3. **Test** your changes and make sure existing tests pass.
4. **Update documentation** if your change affects behavior or usage.
5. **Open a pull request** using the PR template and link any related issues.

### Commit messages

Write clear, descriptive commit messages focused on *why*. We recommend
[Conventional Commits](https://www.conventionalcommits.org/) where practical:

```
feat: add per-document expiry to data rooms
fix: handle empty permission set on share
docs: clarify audit-log export steps
```

### Pull requests

- Keep PRs focused — one logical change per PR is easier to review.
- Fill out the PR template completely.
- Make sure CI passes before requesting review.

## Reporting security issues

**Do not** open public issues for security vulnerabilities. Please follow our
[Security Policy](./SECURITY.md) instead.
