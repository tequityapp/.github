# Security Policy

The Tequity team takes the security of our software and services seriously —
especially given that Tequity handles confidential documents. We appreciate the
efforts of security researchers and the community in helping keep our projects
and users safe.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately using one of the following channels:

- **GitHub Security Advisories** (preferred): open a private report via the
  **Security** tab of the affected repository.
- **Email**: [security@tequity.site](mailto:security@tequity.site)

Please include as much as you can:

- The type of issue (e.g. injection, authentication bypass, data exposure).
- The affected repository, version, or commit.
- Step-by-step instructions to reproduce the issue.
- Proof-of-concept exploit code, if available.
- The impact of the issue and how an attacker might exploit it.

## Our Commitment

When you report a vulnerability, we will:

1. **Acknowledge** receipt of your report within **3 business days**.
2. **Investigate** and provide an initial assessment of severity and validity.
3. **Keep you informed** of our progress toward a fix.
4. **Notify you** when the issue is resolved.

We ask that you give us a reasonable amount of time to address the issue before
any public disclosure, and that you make a good-faith effort to avoid privacy
violations, data destruction, and service interruption during your research.

## Supported Versions

Unless a specific repository states otherwise, security updates are provided for
the latest released version on the default branch.

## Recognition

We're grateful to everyone who helps keep Tequity secure. With your permission,
we're happy to credit you for responsibly disclosed reports.
